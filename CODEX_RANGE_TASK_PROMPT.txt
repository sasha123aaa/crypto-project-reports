Ты работаешь с репозиторием crypto-project-reports.

Прочитай файл RANGE_LOGIC_SPEC.md и используй его как источник истины для блока «Быстрый теханализ».

Твоя задача:
1. Найти, где в репозитории сейчас реализован quick TA.
2. Найти, используется ли там реальная range logic или только упрощенный слой.
3. Если логика не подключена напрямую, подключить ее корректно.
4. Убрать зависимость от BTC static mode.
5. Сделать так, чтобы символ для range logic брался из текущего проекта отчета.
6. Не ломать текущий UI.
7. Не менять смысл логики диапазонов.
8. Не заменять range logic никакими индикаторами, EMA, RSI, close/open candle logic и другими упрощениями.
9. Если не хватает данных или что-то не готово, использовать честный neutral fallback.

Что важно:
- направление по ТФ должно строиться через previewRange[4]
- bullish если true
- bearish если false
- neutral если previewRange нет или он невалиден
- текущие параметры логики должны остаться:
  - correctionPct = 0.3
  - maxRects = 20
  - minBars = 7
