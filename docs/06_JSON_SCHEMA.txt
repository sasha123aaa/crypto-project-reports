ЕДИНАЯ JSON-СХЕМА ДЛЯ ОТЧЕТОВ ПО КРИПТОПРОЕКТАМ

Главное правило:
все отчеты должны возвращаться в одном и том же верхнеуровневом формате.

Обязательные блоки:
1. meta
2. hero
3. market
4. technical_bias
5. executive_summary
6. profile
7. about
8. tokenomics
9. financials
10. capital
11. users
12. liquidity
13. valuation
14. narrative
15. risks
16. watchlist
17. final_verdict
18. charts
19. sources

Правила по метрикам:
— каждая метрика должна иметь:
value
formatted
status
source

Допустимые статусы:
live
calculated
partial
manual
unavailable

Правила:
— если данных нет, блок не удалять
— если метрика не подходит проекту, это писать прямо
— если данные спорные, это писать прямо
— не придумывать цифры
— сначала факт, потом почему это важно, потом вывод
— риски всегда отдельно
— итоговый вывод не должен повторять цифры
